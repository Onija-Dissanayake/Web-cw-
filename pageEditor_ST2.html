Splash Page

Technical Description

The Splash page acts as an engaging entry point for the website. It utilizes CSS Flexbox to vertically and horizontally center the content over a full-viewport background image. I implemented a custom CSS animation (`@keyframes spin`) to create a smooth, infinitely rotating loading indicator.

JavaScript was extensively used to handle the page's lifecycle. A `setInterval` function powers the 4-second countdown timer displayed to the user. Once the timer reaches zero, or if the user clicks the "Skip Intro" button, a JavaScript function triggers a CSS opacity transition to fade out the page, followed by a `setTimeout` function that redirects the browser to the main home page.

Accessibility

Text readability against the background image is maintained by utilizing strong CSS text-shadows and highly contrasting white fonts. A clear, easily clickable "Skip Intro" button ensures users who rely on keyboards or prefer not to wait can bypass the timed redirect immediately.

AIS (Action Impact Score) Page

Technical Description

The AIS page is an interactive tool designed to calculate a user's environmental impact based on their willingness to commit to certain actions. The layout uses CSS Flexbox to arrange the action cards evenly across two sections, incorporating hover effects (`transform` and `box-shadow`) to encourage user interaction.

The core functionality is driven by JavaScript. Event listeners are attached to each card using 'querySelectorAll'. When a card is clicked, a 'selected' CSS class is toggled. The script dynamically adds or subtracts predefined impact points based on the card's ID, updates the total score text, and calculates a percentage. This percentage is then applied to the inline `width` style of a custom CSS progress bar to provide immediate visual feedback.

Accessibility

The page uses clear, descriptive headings and text within each card so users know exactly what action they are selecting. The visual state of a selected card changes significantly (background color and border changes) to ensure the selection state is visually obvious.

Content Page (Student 2)

Technical Description

My content page focuses on "Threats to Biodiversity," exploring key topics like Habitat Destruction, Climate Change, Pollution, and Overexploitation. The page uses a clean, linear layout structured with semantic HTML5 tags ('(section)'', '(h3)', '(p)'').

To enhance navigation, I created a glassmorphism-styled internal navigation menu at the top of the page. This uses HTML anchor links (`href="#id"`) mapped to the IDs of the specific content sections below. I also implemented a fixed "Go to Top" button utilizing CSS `position: fixed`, which hovers in the bottom-right corner of the viewport to allow quick navigation back to the menu regardless of scroll depth.

Accessibility

Semantic heading hierarchies are strictly maintained for screen readers. High-quality images illustrating the threats are accompanied by descriptive `alt` text to ensure visually impaired users understand the context of the visuals. Focus states and color contrasts on the navigation pills were optimized for readability.

Challenges and Lessons Learned

A major challenge I faced was writing the JavaScript for the AIS page. Ensuring that points were accurately subtracted when a user deselected a card—and preventing the progress bar from breaking if multiple rapid clicks occurred—required careful logic and state management.

Additionally, timing the visual fade-out effect on the Splash page with the actual page redirect (`window.location.href`) took some trial and error with `setTimeout` to ensure the transition felt smooth and didn't redirect before the fade finished. These challenges greatly improved my understanding of DOM manipulation and JavaScript timing events.

Compliance

All pages I developed follow modern web publishing standards. HTML5 semantic elements were used to structure the DOM effectively.

Visual styling is handled via external CSS (`global.css`), maintaining a strict separation of concerns. The code has been validated against W3C standards to ensure cross-browser compatibility and structural integrity.
