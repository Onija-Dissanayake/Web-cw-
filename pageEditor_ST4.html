User Profile Page

Technical Description

The User Profile page was developed using HTML, CSS and JavaScript to create an interactive profile creation system. Instead of using a traditional form, JavaScript prompt() functions were used to collect user information step by step. This allowed the profile information to be displayed dynamically as the user completed each prompt.

The page structure was built using semantic HTML elements such as header, navigation, main, section and footer. These elements help organise the content and maintain consistency with the global website layout. JavaScript DOM manipulation was used to insert the user's answers directly into the profile output area.

A progress indicator was implemented to show how much of the profile had been completed. Conditional logic allowed users to skip certain prompts or return to previously skipped questions. This improves usability and allows flexibility during profile creation.

Accessibility

Accessibility was considered by ensuring clear instructions were provided for all prompts. The page uses a structured heading hierarchy so screen readers can understand the page layout. Colour contrast and readable fonts were used to ensure the content is accessible to users with visual impairments. Navigation elements are also consistent with the rest of the website to improve usability.

Sitemap Page

Technical Description

The Sitemap page was implemented using Scalable Vector Graphics (SVG) to visually represent the structure of the website. Each webpage is represented using rectangular nodes connected with lines to illustrate the hierarchical structure of the site.

SVG elements such as rectangles, lines and text were used to construct the diagram. Each page node was wrapped in a clickable link so users can navigate directly to the selected page. This allows the sitemap to function as both a visual diagram and a navigation tool.

The SVG viewBox property was used to make the sitemap responsive. This ensures that the diagram scales correctly across different screen sizes while maintaining readability. CSS styling was also used to apply hover effects to nodes so users can easily identify clickable elements.

Accessibility

Accessibility was addressed by providing descriptive text labels inside each node. Title elements were added so screen readers can describe the purpose of each link. Keyboard navigation was also considered to ensure users can tab through sitemap links easily. Clear contrast between nodes and background colours improves readability.

Content Page (Student 4)

Technical Description

The content page was created to present information related to the Sustainable Development Goal "Life on Land". The page structure uses semantic HTML elements including sections, headings, paragraphs and images to organise the information clearly. Internal navigation links were placed at the top of the page to allow users to jump directly to specific sections.

CSS styling from the global stylesheet was used to maintain consistent colours, typography and layout across the website. Spacing and alignment were controlled using margin and padding properties. Images were added to visually support the written information and make the page more engaging for users.

A "Go to Top" button was included to improve navigation on longer pages. This feature allows users to quickly return to the top of the page without scrolling manually.

Accessibility

Accessibility was addressed by using clear heading structures and meaningful text descriptions. All images include alternative text so screen readers can interpret their content. Colour contrast between background and text was carefully selected to ensure readability for all users.

Challenges and Lessons Learned

One of the main challenges during development was implementing the interactive User Profile page. Handling multiple prompts and updating the profile dynamically required careful use of JavaScript logic and DOM manipulation. Another challenge was designing the SVG sitemap layout so that all pages were clearly connected and visually balanced.

Through solving these challenges, I gained a better understanding of how HTML, CSS and JavaScript work together to create interactive web pages. I also learned the importance of consistent design and usability across different pages of a website.

Compliance

The pages developed follow the guidelines for web publishing and accessibility. Semantic HTML elements were used to structure content properly and improve search engine indexing. External CSS stylesheets were used to separate design from structure, making the code easier to maintain.

Images include alternative text descriptions and links are clearly labelled to improve accessibility. The pages were also tested using HTML validation tools to ensure they meet web standards and contain no structural errors.
