Feedback Page

Technical Description

The Feedback page is a multi-functional layout developed using HTML, CSS, and JavaScript. It serves as both a directory for community programs and a platform for user interaction through a feedback form. The upper section features a filtering layout and interactive program cards with CSS scale and box-shadow hover effects.

The form section was built using semantic HTML fieldsets and input types (email, tel, date). JavaScript was heavily utilized to provide client-side form validation. This includes checking for empty fields, verifying email format, and displaying dynamic error messages.

Additionally, an event listener was implemented on the textarea to create a real-time character counter that updates as the user types, restricting the message to 150 characters.

Team Page

Technical Description

The Team page uses advanced CSS techniques to present the project contributors. The layout is built using CSS Flexbox, ensuring the profile cards wrap and align symmetrically across different screen sizes.

A modern "glassmorphism" effect was achieved on the cards using a combination of semi-transparent RGBA backgrounds, borders, and the `backdrop-filter: blur()` property. Complex CSS hover interactions were implemented, including a dark overlay that slides up seamlessly using `transform: translateY()` and `opacity` transitions to reveal the student's role and contributions.

Scalable Vector Graphics (SVG) were embedded directly into the HTML to render social media icons, ensuring they remain crisp on all displays. Fill colors on these SVGs transition smoothly on hover.

Accessibility

Alternative text (`alt` attributes) was provided for all team member portrait images. The contrast ratio between the text on the dark overlay and the background was maximized to ensure readability. Semantic heading levels (h2, h3, h4) were strictly maintained.

Challenges and Lessons Learned

One of the primary challenges was implementing the client-side JavaScript validation for the feedback form. Ensuring the error messages appeared dynamically without resetting the page required a deep dive into event listeners and the `e.preventDefault()` method.

Another challenge was perfecting the glassmorphism and overlay hover effects on the Team page. Balancing the z-index so that social icons remained clickable while the text overlay appeared required precise CSS structuring. Through this, I significantly improved my understanding of CSS positioning, transitions, and DOM manipulation.

Compliance

The developed pages comply with standard web publishing guidelines. Semantic HTML5 tags were utilized properly to construct the document tree, enhancing both SEO and accessibility.

Design and structure were strictly separated by using external CSS stylesheets. The code has been checked against W3C validators to ensure no structural anomalies exist, and all media elements are supported by accessible fallback text.
