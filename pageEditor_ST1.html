Home page

Technical Description

The Home page uses the shared semantic HTML template (`<header>`, `<nav>`, `<main>`, `<footer>`). I implemented embedded CSS to style page-specific elements without overriding `global.css`. I utilised CSS Flexbox for the central information cards to ensure they sit side-by-side, and CSS Grid (`grid-template-columns: repeat(4, 1fr)`) to create the structured 4-column layout for the bottom gallery links. CSS transitions were applied to images to create a scale effect on hover.

Accessibility

Accessibility was addressed by ensuring high contrast between the white text and dark green backgrounds. Every image utilizes descriptive `alt` attributes (e.g., "Forest conservation") so screen readers can interpret the visual context. Semantic structure is strictly maintained.

Gallery Page

Technical Description

The Gallery uses a 2-column CSS Grid layout for the thumbnails. The core functionality relies on JavaScript interacting with custom HTML `data-` attributes (`data-title`, `data-desc`, `data-img`). When a thumbnail is clicked, JavaScript extracts these attributes and dynamically populates a hidden modal `<div>`. The modal's visibility is toggled by adding/removing a `.show` CSS class. DOM manipulation is also used to satisfy the user customization requirement, allowing users to alter the modal's background color and font family via button clicks.

Accessibility

Alt text is provided for all gallery thumbnails. The JavaScript modal includes a highly visible close button (`×`) and an event listener that allows users to close the modal by clicking anywhere outside of it, ensuring users don't get trapped in the extended view.

Content Page

Technical Description

This page features long-form text structured sequentially with `<h1>`, `<h2>`, and `<h3>` tags. I implemented an internal navigation menu using anchor links (`href="#id"`) that map to specific `id` attributes on the `<section>` tags. A "Go to Top" button was created using CSS `position: fixed` to keep it accessible in the viewport regardless of scroll depth. Images were styled centrally with borders and box-shadows to fit the visual identity.

Accessibility

The logical heading hierarchy makes document navigation seamless for screen readers. The internal navigation list offers an immediate skip-link alternative to scrolling, enhancing usability for keyboard navigators.

Challenges and Lessons Learned

Coordinating the `global.css` file across the team was an early challenge. Because multiple team members needed to link to the same stylesheet, we had to ensure our Git workflow was solid. Resolving minor merge conflicts when pulling changes in VS Code taught me the importance of clear communication and modular CSS design. Technically, managing the JavaScript modal overlay on the Gallery page without blocking the underlying page elements required careful tuning of `z-index` and `opacity` transitions.
